# 04. 研究テーマと方向性

## 4.1 研究テーマ

### 統一テーマ

> **Organizational Causal Twin (OCT): LLMマルチエージェントシミュレーションによるDAG-free因果推論フレームワーク**

### 上位リサーチクエスチョン

> LLMマルチエージェントシミュレーションにより組織環境をツインとして構築し、そのツインへの介入を通じて、事前のDAG定義なしに、知識想起（LLMへの直接質問）では検出できない因果関係を発見できるか？

### サブRQ

- **RQ1:** OCTシミュレーションは、LLMへの直接質問（Mode R）では得られない創発的イベントを生成するか？
- **RQ2:** 創発的イベントの因果連鎖は、ステップバイステップのエージェント間相互作用を通じてのみ形成されるものか？
- **RQ3:** OCTによる因果効果推定は、従来のDAGベース推定と整合するか？整合しない場合、その差分はどのような因果構造に起因するか？

## 4.2 提案の独自性

### 先行研究との位置づけ

```
              事前定式化の程度
        高い ←─────────────────→ 低い

従来DAG     ProCI      ABM        OCT
(完全定式化)  (DAG+変数追加) (ルール定義)  (環境ツイン)
                                     ↑
                          CALM-DT, DTCFが理論的基盤を提供
                          Generative Agentsが創発の実証先例
```

| 先行研究 | 差分 |
|---------|------|
| CALM-DT | 物理システム対象。因果推論への接続なし → OCTは因果推論パラダイムとして定位 |
| DTCF | 理論フレームワーク。LLM実装なし → OCTが実装を提供 |
| ProCI/VIGOR+ | DAG内での変数追加 → OCTはDAGの事前定義そのものを回避 |
| Generative Agents | 社会行動の観察 → OCTは因果効果の推定を目的とする |

### 独自性の核心

**「DAGの事前定義」という因果推論の最大のボトルネックを、LLM環境シミュレーションで迂回する」**

これは特定ドメインに限定されない一般的な方法論的提案。内部統制は最初の実証ドメイン。

## 4.3 他の方向性との関係

方向Dを主軸としつつ、A〜Cの知見は副次的に取り込む。

| 方向 | OCTとの関係 |
|------|-----------|
| A: LLM判断の信頼性保証 | OCT内のエージェントの行動決定の信頼性検証として内包 |
| B: LLM×因果推論で統制評価 | OCTの一応用ケース。DAGベース推定との比較対照として使用 |
| C: A+Bの統合 | OCTフレームワークが自然に統合する |

## 4.4 想定される貢献

| 種別 | 貢献 |
|------|------|
| 方法論 | DAG-freeの因果推論パラダイム（シミュレーション→介入→観察）の提案 |
| 技術 | OCTのアーキテクチャ設計、創発性検証プロトコルの実装 |
| 実証 | 内部統制ドメインでの創発的因果関係の発見 |
| 理論 | DTCFをLLMマルチエージェントシミュレーションに拡張する理論的整理 |

## 4.5 想定されるリスク

| リスク | 影響 | 対策 |
|-------|------|------|
| LLMが環境を忠実に再現できない | OCTの推定結果が信頼できない | DTCFの検証アーキテクチャ適用。factual fidelityの検証 |
| 創発が「知識想起」と区別できない | OCTの独自価値が消失 | 三層検証プロトコル（創発性/経路依存性/相互作用必要性テスト） |
| 計算コストが高い | 大規模実験が困難 | 小規模モデル、限定的なエージェント数、ステップ数最適化 |
| LLMバイアスが結果に影響 | ステレオタイプ的な結果 | 複数LLMでのrobustness check |
| 理論的正当化が不十分 | 学術的受容が困難 | DTCFの枠組みでの位置づけ、限界の明示 |
