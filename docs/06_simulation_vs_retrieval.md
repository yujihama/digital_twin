# 06. シミュレーション vs 知識想起の判別

## 6.1 問題の定義

OCTの価値を主張するには、シミュレーション出力が「LLMに聞いただけ」（Mode R）と区別できなければならない。

| | Mode R（知識想起） | Mode S（シミュレーション） |
|---|---|---|
| 入力 | 「Xが起きたらどうなるか？」 | 環境にXを介入し、ステップごとに実行 |
| 因果パス | 既知（学習データ由来）のみ | 既知＋創発的（相互作用から出現） |
| 時間展開 | なし（一括回答） | あり（状態遷移の連鎖） |
| エージェント間相互作用 | なし | あり |
| 確率的変動 | 低い | 高い（経路依存性） |
| 出力の質 | 「ありがちな結果」の想像 | 「この環境での具体的な結果」の生成 |

## 6.2 創発が生じるための4条件

先行研究（Generative Agents, 群知能シミュレーション等）から抽出。

| 条件 | 内容 | OCTでの実装 |
|------|------|-----------|
| **C1: 複数エージェントの相互作用** | A→B→C→Aのフィードバックループ | Agent Layer内の直接的コミュニケーション |
| **C2: 状態の時間的蓄積** | 各ステップの結果が次の入力になる | メモリ + 環境状態の更新 |
| **C3: 行動規則の非完全指定** | LLMの「判断の余地」が創発の源泉 | 原理ベースのプロンプト（ルールベースではなく） |
| **C4: 環境からのフィードバック** | 行動→環境変化→行動（閉ループ） | Environment Stateからの部分観測配信 |

## 6.3 三層検証プロトコル

### Layer 1: 創発性テスト（Emergence Test）

同一の介入に対し Mode R（直接質問）と Mode S（OCTシミュレーション）の出力を比較。

```
定量指標:
  Emergence Ratio = |Mode Sのみのイベント| / |全イベント(Mode S)|
  
定性評価:
  創発イベントの「もっともらしさ」をドメイン専門家が評価
```

### Layer 2: 経路依存性テスト（Path Dependency Test）

同一条件でN回シミュレーションし、結果の分散を検証。

```
判定:
  CV(Y) > 閾値 → 経路依存性あり → 真のシミュレーション
  CV(Y) ≈ 0 → 決定論的 → Mode Rへの退化の疑い
```

### Layer 3: 相互作用必要性テスト（Interaction Necessity Test）

エージェント間の情報交換を遮断して結果を比較。

```
判定:
  |Y_full - Y_isolated| が有意 → 相互作用が因果的に寄与
  |Y_full - Y_isolated| ≈ 0 → 相互作用は装飾的（Mode Rと同等）
```

## 6.4 検証結果の報告テンプレート

```
OCT実験報告:
  介入: [介入内容]
  シミュレーション: N回実行、各Tステップ

  Layer 1:
    Mode R予測リスク: X件
    Mode Sイベント: Y件
    創発イベント: Z件 (Emergence Ratio = Z/Y)

  Layer 2:
    結果変数の分布: μ, σ, CV
    質的に異なるシナリオ: M種類

  Layer 3:
    |Y_full - Y_isolated|: [値]
    相互作用の因果的寄与: [有意/非有意]
```
