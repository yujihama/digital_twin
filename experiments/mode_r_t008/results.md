# T-008: Mode R 予測 vs Mode S 観測 — Layer 1 創発性テスト

> **目的**: Mode R（LLM知識想起）で予測可能な事象と、Mode S（シミュレーション）でのみ観測された事象を比較し、Emergence Ratio を算出する。

## 実験条件

| 項目 | 値 |
|---|---|
| Mode R モデル | gpt-4.1-mini |
| Mode R temperature | 0.8 |
| Mode S 実験 | exp004（閾値500万円） vs exp003c（閾値20万円） |
| Mode R 予測数 | 18 |
| Mode R プロンプト | 承認閾値20万→500万の影響予測（6観点） |

## Mode R 予測とMode S 観測の突合

### 分類基準

- **Mode R 的中**: Mode R がexp004で実際に観測された現象を予測していた
- **Mode R 不的中**: Mode R が予測したがexp004では観測されなかった
- **Mode S のみ（創発）**: Mode S でのみ観測され、Mode R では予測されなかった

## Emergence Ratio 算出

### 定義

```
Emergence Ratio = |Mode S のみで観測された事象| / |Mode S の全観測事象|
```

### 算出

| 分類 | 件数 | 該当 |
|---|---|---|
| Mode S 全観測事象 | 9 | S-001〜S-009 |
| 予測可能（Mode R 的中） | 2 | S-001, S-003 |
| 部分的に予測可能 | 1 | S-002 |
| **Mode S のみ（創発）** | **6** | **S-004, S-005, S-006, S-007, S-008, S-009** |

```
Emergence Ratio = 6 / 9 = 0.667 (66.7%)
```

**保守的算出**（部分的中を「予測可能」に含める場合）:
```
Emergence Ratio (conservative) = 6 / 9 = 0.667 (66.7%)
```

**厳密算出**（部分的中を「創発」に含める場合）:
```
Emergence Ratio (strict) = 7 / 9 = 0.778 (77.8%)
```

## 創発事象の分析

### 高確度の創発事象

1. **S-004: vendor_e間接活性化** — 承認ルール変更はvendor_eに直接関係しないにもかかわらず、注文増加を通じてwait 8→1の劇的な行動変化を引き起こした。Mode Rは取引先の行動変化を3件予測したが、いずれも「直接的な影響」を想定しており、「承認→buyer→vendor」という間接的な因果連鎖を捕捉できなかった。

2. **S-005: 統制品質の維持** — Mode R（R-013）は「三者照合頻度が減少し統制の空白が生じる」と予測したが、実際にはdeviation_count=0で三者照合100%が維持された。LLMエージェントは統制緩和下でも誠実な取引行動を選択し続けた。これはLLMの行動特性に依存する創発的結果である。

3. **S-006: 重複請求エラー** — フロー加速による処理密集が、vendor_eの重複請求試行を引き起こした。「統制緩和→フロー加速→処理衝突」という二次的因果連鎖は、Mode Rの知識想起では予測されなかった。

4. **S-009: 経理活性化** — Mode R（R-010）は「支払処理負荷が減る」と予測したが、実際にはpay_order 20→24で増加した。承認ボトルネック解消が経理の待ち時間を減らし、より多くの支払処理を可能にした。

### OCTフレームワークへの示唆

Emergence Ratio = 66.7〜77.8% は、Mode S（シミュレーション）がMode R（知識想起）では検出できない因果関係を有意に発見できることを示す。特に：

- **間接的因果連鎖**（S-004, S-009）: 承認→buyer→vendor→accountantの連鎖的波及は、知識想起では捕捉困難
- **逆方向の結果**（S-005, S-009）: Mode Rの常識的予測と逆の結果が出現
- **二次的副作用**（S-006）: フロー加速が引き起こす処理衝突は動的シミュレーションでのみ検出可能
- **定量的効果**（S-007, S-008）: 具体的な数値変化はシミュレーションでのみ測定可能

## 制限事項

1. **単一seed**: seed=42のみでの結果であり、一般化には複数seedでの再現が必要（T-016）
2. **単一LLM**: gpt-4.1-miniのみ。異なるLLMでのMode R予測との比較が必要
3. **環境制約**: vendor_eが1社のみの環境であり、R-008（取引先選定変化）等の検証が本質的に不能
4. **Mode R prompt依存**: プロンプトの設計によりMode R予測が変わる可能性がある
5. **判定の主観性**: Mode R予測とMode S観測の対応付けには解釈の余地がある
